# Інструкція з розгортання у виробничому середовищі (Production)

Цей документ призначений для DevOps-фахівців та Release-інженерів для розгортання проєкту **GamerIS**.

## 1. Вимоги до апаратного забезпечення
Оскільки проєкт є статичним веб-додатком (SPA), основне навантаження припадає на сторону клієнта (браузер). 

**Мінімальні вимоги до сервера збірки (CI/CD або Build machine):**
- **Архітектура**: x86_64 або ARM64.
- **CPU**: 1 Core 1.5 GHz.
- **RAM**: 1 GB (мінімум для стабільної роботи `npm install` та `vite build`).
- **Диск**: 500 MB вільного місця.

## 2. Необхідне програмне забезпечення
Для підготовки та розгортання коду:
- **Node.js**: v18.18.0 або новіша.
- **npm**: v9.x або новіша.
- **Git**: для отримання вихідного коду.
- **Браузер**: Chrome 100+, Firefox 100+, Safari 15+ (для клієнтів).

## 3. Налаштування мережі
- **Протокол**: HTTPS (порт 443).
- **Доступ до API**: Машина клієнта повинна мати доступ до доменів `*.google.com` та `*.googleapis.com` для взаємодії з Google Sheets та Google Apps Script.
- **CORS**: Налаштовано на стороні Google Apps Script для дозволу запитів з домену розгортання.

## 4. Конфігурація серверів та СУБД
- **Веб-сервер**: GitHub Pages (статичний хостинг). Конфігурація серверної частини (Nginx/Apache) не потрібна, оскільки хостинг є керованим сервісом.
- **СУБД (Google Sheets)**: 
  - База даних є зовнішнім SaaS-рішенням.
  - Необхідно переконатися, що Google-таблиця має права доступу "Anyone with the link can view" (або закритий доступ через сервісний аккаунт).
  - ID таблиці та URL скрипта мають бути прописані в константах середовища додатка.

## 5. Розгортання коду
Процес розгортання автоматизовано. Виконайте наступні команди:

1.  **Отримання коду**:
    ```bash
    git clone [https://github.com/vadimshevc/gamer-is.git](https://github.com/vadimshevc/gamer-is.git)
    cd gamer-is
    ```
2.  **Підготовка збірки**:
    ```bash
    npm install
    npm run build
    ```
3.  **Публікація**:
    ```bash
    npm run deploy
    ```
    *Примітка: команда `deploy` використовує пакет `gh-pages` для перенесення вмісту папки `dist` у відповідну гілку репозиторію.*

## 6. Перевірка працездатності (Health Check)
Як зрозуміти, що все працює правильно:
1.  **Завантаження інтерфейсу**: Відкрийте URL проєкту. Сторінка лендінгу повинна завантажитися без помилок у консолі (F12).
2.  **Зв'язок з БД**: Перейдіть до розділу каталогу товарів. Якщо список товарів відображається — зв'язок з Google Sheets встановлено успішно.
3.  **Мережеві запити**: У вкладці Network браузера запити до `script.google.com` повинні мати статус `200 OK`.

## 🤖 Автоматизація рутинних операцій
Для зручності розгортання та розробки у папці `docs/scripts` додано автоматизаційні скрипти:
- `dev_start`: Встановлення залежностей та запуск dev-сервера.
- `prod_deploy`: Повний цикл збірки (lint -> build -> deploy).

## 🐳 Контейнеризація та CI/CD
Для проєкту налаштовано сучасні інструменти автоматизації:
- **Docker**: Додано `Dockerfile` для створення легкого образу на базі Nginx. Запуск: `docker-compose up`.
- **CI/CD**: Налаштовано GitHub Actions (папка `.github/workflows`), що автоматично перевіряє код лінтером та виконує збірку при кожному пуші в `main`.
