# 🕹️ GamerIS Pro — Інформаційна система магазину

## 📝 Опис проєкту
Вебдодаток для автоматизації геймерського магазину: каталог товарів, розрахунок вартості в кошику та перевірка залишків. Проєкт підготовлено як базу для бакалаврської роботи.

## 🚀 Стек технологій
- **React 18 + Vite** (Frontend)
- **Tailwind CSS** (Стилізація)
- **Vitest** (Автоматизоване тестування)

## 📂 Структура репозиторію
- `src/` — Вихідний код програми.
- `src/utils/` — Бізнес-логіка та UNIT-тести.
- `public/` — Статичні ресурси.
- `.gitignore` — Налаштування ігнорування системних файлів.
- `LICENSE` — Ліцензія MIT.

## 🛠️ Як запустити
1. `npm install`
2. `npm run dev`
3. Запуск тестів: `npx vitest run --coverage`

## 📝 Стандарти документування
Всі функції та компоненти мають бути задокументовані за стандартом **JSDoc**. 
Обов'язкові теги: `@param` для вхідних даних та `@returns` для опису результату.

# 🛠 Інструкція для нового розробника (Quick Start)

Цей посібник допоможе вам налаштувати середовище розробки та запустити проєкт **GamerIS** на комп'ютері з щойно встановленою операційною системою.

## 1. Встановлення необхідного ПЗ
Перш за все, необхідно встановити базові інструменти:

1.  **Git**: [Завантажити та встановити](https://git-scm.com/downloads). (Потрібен для клонування коду).
2.  **Node.js** (рекомендовано версію LTS): [Завантажити та встановити](https://nodejs.org/). (Включає менеджер пакетів **npm**).
3.  **VS Code**: [Завантажити та встановити](https://code.visualstudio.com/). (Рекомендований редактор коду).

## 2. Клонування репозиторію
Відкрийте термінал (або Git Bash) та виконайте команду:
`git clone [https://github.com/vadimshevc/gamer-is.git](https://github.com/vadimshevc/gamer-is.git)`
`cd gamer-is`

## 3. Налаштування середовища та залежностей
Встановіть усі необхідні бібліотеки для роботи проєкту: `npm install`

## 4. База даних та API
Проєкт використовує Google Sheets як базу даних через Google Apps Script API.

- Локальна БД: Не потребує встановлення (SQL/NoSQL сервер не потрібен).
- Конфігурація: Переконайтеся, що у файлах проєкту вірно вказано URL вашого розгорнутого Google Apps Script (зазвичай це вже налаштовано в константах).

## 5. Запуск проєкту у режимі розробки
- Для запуску локального сервера з підтримкою "гарячого оновлення" (Hot Reload): `npm run dev`
- Після запуску проєкт буде доступний у вашому браузері за адресою: `http://localhost:5173`

## 6. Базові команди та операції
- `npm run dev` — запуск сервера розробки.
- `npm run build` — створення оптимізованої збірки для продакшну (папка dist).
- `npm run lint` — запуск статичного аналізатора коду (перевірка помилок стилю).
- `npm run deploy` — автоматичне розгортання поточної збірки на GitHub Pages.
